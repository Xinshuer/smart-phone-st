# ComfyUI 工作流 / Prompt 接入指南

给手机插件的 prompt 注入逻辑参考。**3 套工作流，每套对应一个底模**，用户在酒馆里手动选用哪个工作流文件。

---

## 1. 三个工作流文件

| 文件 | 底模 | 架构 | LoRA | 适用场景 |
|---|---|---|---|---|
| `comfyUI生图工作流-pony-realism.txt` | `ponyRealism_V22.safetensors` | SDXL Pony | Realism Enhancer + Amateur Style | **默认首选**：欧美写实，明亮自然光，amateur 手机感，最适合 SillyTavern 自拍/日常 |
| `comfyUI生图工作流-noobai-vpred.txt` | `noobaiXLNAIXL_vPred10Version.safetensors` | SDXL Illustrious vPred | NOOB vPred Detailer @ 0.5 | 亚洲/东亚向，配合 RescaleCFG 后亮调可用，偏暧昧情境时仍有暖调氛围优势 |
| `comfyUI生图工作流-majicmix.txt` | `majicmixRealistic_v7.safetensors` | **SD 1.5** ⚠️ | 暂无（需找 SD 1.5 base 的 LoRA） | 写实美女向，原生 768×1152，作者推荐 adetailer 修脸（已配 FaceDetailer） |

## 3. Prompt 模板（直接复制使用）

每个模型的正负面 prompt 格式完全独立，不能跨用。下面给出可拷贝的结构 + 完整示例。

### 3.1 🟦 Pony Realism V22（默认主用 / 欧美写实）

**正面 prompt 结构**：
```
score_9, score_8_up, score_7_up, photo, amateur, film grain, <人物描述>, <场景>, <姿势/动作>, <光照>
```

**负面 prompt**（直接复制）：
```
score_4, score_5, score_6, lowres, worst quality, low quality, bad anatomy, bad hands, missing fingers, extra fingers, deformed, blurry, watermark, text, signature, censored, mosaic
```

**完整示例**：
```
score_9, score_8_up, score_7_up, photo, amateur, film grain, 1girl, solo, blonde hair, blue eyes, large breasts, mirror selfie, bathroom, holding smartphone, naked, looking at mirror, natural light from window, modern apartment
```

**约定**：
- 用 `female`/`male` 而不是 `woman`/`man`
- 用 Danbooru 风格标签
- 单标签权重 ≤ 1.5，不要 `(xxx:2)`
- amateur 风格触发词：`photo`, `amateur`, `film grain`, `grainy`, `2000s nostalgia`, `webcam photo`, `flash`
- 想加亚洲面孔：在人物描述里加 `asian, asian girl`

---

### 3.2 🟥 NoobAI vPred 1.0（动漫 / 暧昧 / 配 RescaleCFG 后亮调可用）

> 工作流已配置 `RescaleCFG 0.7 + cfg 7.0`，浴室/淋浴/户外场景**已可正常出亮调图**。
> 卧室/室内 prompt 中的 `soft light`/`dim light` 词会被尊重，仍出暖橙暗调（这是符合预期的）。

#### NSFW 写实风格

**正面 prompt 结构**：
```
masterpiece, best quality, newest, absurdres, highres, real photo, photorealistic, raw photo, photo of a real girl, detailed skin, sharp focus, bright, well-lit, daylight, high-key lighting, natural lighting, nsfw, <人物描述>, <场景>, <姿势/动作>, <光照>
```

**负面 prompt**（**不含 nsfw**，包含抑制暗调和 UI 污染的标签）：
```
worst quality, old, early, low quality, lowres, signature, username, logo, bad hands, mutated hands, mammal, anthro, furry, ambiguous form, feral, semi-anthro, dark, dim, low light, underexposed, monochrome, dark room, app interface, status bar, ui, app screenshot, phone screen frame, social media overlay
```

#### SFW 写实风格（把 `nsfw` 换成 `safe`，负面开头加 `nsfw,`）

**正面**：
```
masterpiece, best quality, newest, absurdres, highres, real photo, photorealistic, raw photo, photo of a real girl, detailed skin, sharp focus, bright, well-lit, daylight, high-key lighting, natural lighting, safe, <人物描述>, <场景>, <姿势/动作>
```

**负面**：
```
nsfw, worst quality, old, early, low quality, lowres, signature, username, logo, bad hands, mutated hands, mammal, anthro, furry, ambiguous form, feral, semi-anthro, dark, dim, low light, underexposed, monochrome, dark room, app interface, status bar, ui, app screenshot, phone screen frame, social media overlay
```

#### 动漫风格（去掉 real photo 系列 + bright 系列标签）

**正面**：
```
masterpiece, best quality, newest, absurdres, highres, nsfw, <人物描述>, <场景>, <姿势/动作>
```
（负面去掉 `dark, dim, low light...` 那段，因为动漫场景需要灵活光影）

#### 暗调情绪戏（卧室/烛光/暧昧）

正面 prefix 把 `bright, well-lit, daylight, high-key lighting, natural lighting,` **删除**，自然 prompt 里写 `soft warm light, candlelight, intimate atmosphere` 即可：
```
masterpiece, best quality, newest, absurdres, highres, real photo, photorealistic, raw photo, detailed skin, nsfw, 1girl, solo, asian, lying on bed, naked, looking at viewer, soft warm light, candlelight, intimate atmosphere
```
负面同上去掉 `dark, dim` 系列。

**完整示例（NSFW 写实，亚洲，明亮 selfie）**：
```
masterpiece, best quality, newest, absurdres, highres, real photo, photorealistic, raw photo, photo of a real girl, detailed skin, sharp focus, bright, well-lit, daylight, high-key lighting, natural lighting, nsfw, 1girl, solo, asian, black hair, brown eyes, large breasts, mirror selfie, bathroom, holding smartphone, naked, looking at mirror, natural light from window, modern apartment
```

**关键约定**：
- ⚠️ **NSFW 时不能在负面留 `nsfw`，SFW 时必须在负面加 `nsfw,`**——否则模型自相矛盾
- 想要写实**必须**显式加 `real photo, photorealistic, raw photo` —— 不加默认走动漫
- 想要明亮场景：用上面带 `bright, well-lit, daylight,...` 的 prefix；想要暗调情境则去掉这些词，让 prompt 里的光照标签主导
- ⚠️ **不要写 `instagram, snapchat aesthetic`** —— 会让模型生成完整 app UI 截图（含状态栏、评论框）
- prompt 必须**详细描述脸/服装/灯光**，否则容易出 black-on-black 剪影
- 标签顺序：`<1girl>, <character>, <series>, <artists>, <special>, <general>, <other>`
- 时间标签：`year 2024` / `newest` / `recent` 影响美学
- 质量标签层级（按强度递增）：`worst quality < normal quality < good quality < best quality < masterpiece`

### 3.3 🟨 majicMIX Realistic v7（SD 1.5 / 亚洲写真人像）

**正面 prompt 结构**：
```
Best quality, masterpiece, ultra high res, (photorealistic:1.4), <人物描述>, <场景>, <姿势/动作>, <光照>
```

**负面 prompt**（无 TI 嵌入版，直接复制）：
```
(worst quality, low quality, normal quality:1.4), bad anatomy, bad hands, missing fingers, extra fingers, fewer digits, extra limbs, deformed, mutation, blurry, watermark, text, signature, lowres, jpeg artifacts, cartoon, 3d, anime, cgi
```

**负面 prompt**（下载 TI 嵌入后用更短的版本）：
```
embedding:ng_deepnegative_v1_75t, embedding:badhandv4, (worst quality, low quality:1.4), bad anatomy, bad hands, blurry, watermark, text
```

**完整示例**：
```
Best quality, masterpiece, ultra high res, (photorealistic:1.4), 1girl, solo, asian, black hair, brown eyes, mirror selfie, bathroom, holding smartphone, naked, large breasts, looking at mirror, natural light from window, modern apartment
```

**暗调场景追加（作者钦定）**：在正面追加 `in the dark, deep shadow, low key`

**特殊语法**：
- ComfyUI CLIPTextEncode **支持 A1111 风格权重** `(text:1.4)` —— 不需要换节点
- `BREAK` 关键字默认 ComfyUI 不识别（要用得切到 smZ CLIPTextEncode）

**关于 TI 嵌入**：
- `ng_deepnegative_v1_75t` 和 `badhandv4` 需从 [Civitai](https://civitai.com/models/4629) 手动下载
- 放到 `g:/本地部署/ComfyUI/models/embeddings/`
- 用 `embedding:文件名（不含扩展名）` 引用

## 5. 场景路由建议（重要！手机插件必读）

3 个底模各擅长不同场景，手机插件按上下文选工作流：

| 场景 | 推荐工作流 | 理由 |
|---|---|---|
| **默认 / 不确定** | **Pony Realism** | 80% 场景都合适，amateur 手机随手拍感最强 |
| 欧美/拉丁/混血外貌（自拍 / 户外 / 街拍 / 餐厅 / 客厅） | **Pony Realism** | amateur 风格最像真实欧美手机照片 |
| **亚洲/东亚外貌**（写真级人像质感、自拍、日常） | **majicMIX**（首选） | 实测亚洲面孔表现完胜 NoobAI，写真级清纯亮丽 |
| 动漫/二次元/2.5D 角色 | NoobAI vPred | 默认就是动漫风，2D 表现力最强 |
| 角色想要明显 cinematic 暗调氛围（情感戏） | NoobAI vPred + 加 `real photo, photorealistic, dim lighting` | 它的暗暖调正好成为优势 |
| SFW 普通对话出图 | Pony Realism（默认） | 默认更亮，amateur 风让画面接地气 |

**简单决策树**：
```python
if 角色是亚洲/东亚面孔:
    use majicMIX        # ← 亚洲面孔最佳，不是 NoobAI！
elif 角色是动漫/二次元:
    use NoobAI vPred    # ← 唯一动漫选手
else:  # 欧美/拉丁/混血/默认
    use Pony Realism
```

**重要提醒**：之前以为「亚洲面孔配 NoobAI」是常识——实测错了。**NoobAI 默认走动漫风**（作者它本来就是动漫模型），即便加了写实标签也偏 cinematic 暗调。**亚洲写实人像走 majicMIX 更稳**。

## 6. SillyTavern 占位符替换映射

工作流里的 `%xxx%` 字段会被酒馆替换：

| 占位符 | 哪个工作流支持 | 备注 |
|---|---|---|
| `%prompt%` | 全部 3 个 | **核心注入点**：手机插件构造的正向 prompt |
| `%negative_prompt%` | 全部 3 个 | 负向 prompt |
| `%width%` × `%height%` | 全部 3 个 | majicMIX 用 768×1152，其他用 832×1216 |
| `%steps%` | 全部 3 个 | 都建议 30 |
| `%denoise%` | 全部 3 个 | 文生图固定 1.0 |
| `%scale%` `%sampler%` `%scheduler%` | **不再使用** | 全部硬编码进各工作流，避免酒馆默认值踩坑 |

**关键**：手机插件不要依赖酒馆的 sampler/cfg/scheduler 设置，因为工作流硬编码会忽略这些。

## 7. API 调用示例（手机插件直接 POST）

```python
import json, urllib.request

with open('g:/本地部署/comfyUI生图工作流-pony-realism.txt', encoding='utf-8') as f:
    workflow = json.load(f)

# 替换占位符
workflow['7']['inputs']['text'] = "score_9, score_8_up, score_7_up, photo, amateur, film grain, 1girl, ..."
workflow['8']['inputs']['text'] = "score_4, score_5, score_6, lowres, worst quality, ..."
workflow['9']['inputs']['width'] = 832
workflow['9']['inputs']['height'] = 1216
workflow['10']['inputs']['steps'] = 30
workflow['10']['inputs']['denoise'] = 1.0
workflow['10']['inputs']['seed'] = some_random_seed

req = urllib.request.Request(
    'http://127.0.0.1:8188/prompt',
    data=json.dumps({"prompt": workflow}).encode('utf-8'),
    headers={'Content-Type': 'application/json'}
)
resp = json.loads(urllib.request.urlopen(req).read())
prompt_id = resp['prompt_id']
# 轮询 GET /history/<prompt_id> 检查完成
# 完成后取 ComfyUI/output/PonyRealism_2K_*.png
```

输出文件名前缀：
- Pony Realism → `PonyRealism_2K_*.png`
- NoobAI → `Noobai_2K_*.png`
- majicMIX → `Majic_2K_*.png`

## 8. 已知坑

1. **LoRA 路径必须用反斜杠**：JSON 里写 `Pony\\zy_Realism_Enhancer_v2.safetensors`，不能用正斜杠（ComfyUI 在 Windows 上枚举的就是反斜杠路径）
2. **NoobAI 唯一支持 Euler**：作者明确说其他 sampler "will not work properly"。dpmpp_2m_sde 会出现色彩通道爆掉（V2 实测过）
3. **NoobAI dim 已用 RescaleCFG 修复**：工作流加了 `RescaleCFG 0.7 + cfg 7.0`，浴室/淋浴/户外亮调可用。但仍**不能写 `instagram, snapchat aesthetic`**（会出 UI 截图）；卧室室内要亮调记得用第 3.2 节的「亮调」prefix
4. **majicMIX 是 SD 1.5**：不能套 SDXL/Pony 的 LoRA，分辨率也不一样
5. **FaceDetailer 必须重启 ComfyUI 加载**：ComfyUI 启动时才扫描 custom_nodes，新装节点需重启
6. **特殊字符路径**：`NoobAI-XL (NAI-XL)的LoRA_动漫风格` 这种含中文+全角括号的目录，JSON UTF-8 编码就行，调用 API 时确保 `Content-Type: application/json; charset=utf-8`
